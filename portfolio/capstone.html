Overview

Each Spring and Fall sesmester, the College of Computing and Software Engineering hosts their Hackathon event — where local companies sponsor teams of students all competing to win prizes, recognition, and possible career opportunities. At the time of this project, the current method of managing sponsors, students, and event details required the use of multiple programs and put a large strain on the event coordinators.

Our goal for this project was to create a single web applicaiton that would allow students to register for the event, set up teams, and select their own sponsors. Our system also had adminstrative capabilities such as team management, event and sponsor updates, event scheduling, and more.

Planning Phase

SCREENSHOT OF GANTT CHART W/ POSSIBLE DOWNLOAD LINK

During the planning phase, we quickly realized that our team had complimentary skills. This allowed us to effectively design each element of the application in a way that would ensure that they work together. The requirements given to us by the client were as follows:

  • Students must be able to create accounts and manage their information
  • Students must be able to create and manage their teams
  • Teams must be able to register for sponsors/programs
  • Admin must be able to manage events and sponsor selection
  • Admin must bea able to declare and display winning teams
  • Technical and user documentation must be provided for handoff in Spring 2021
  • Create a Discord server that will serve as an online community (optional)

The only material we were provided to complete this project was a remote server running Ubuntu 18.04. Thankfully, we had all had some degree of experience with Ubuntu, and were therefore able to set up preliminary databases and host our site files with little to no issue. From here, we broke the project up into three milestones:

  1. Account registration — creating a webform and accompanying database that allowed a collection of important data; populate student dashboard
  2. Admin capabilities — creating a separate administrator login and dashboard with required functionalities
  3. Final working solution — finish out base features and create handoff/technical documentation

Each milestone was approximately 4 weeks long, with weekly meetings with our client/sponsor to ensure our project was on the right path.

Development Phase

This was where the real fun began. We immediately began creating wireframes and a style guide for each of our necessary pages. Our client had asked us to include the school's colors — gold and black — and other branding elements into our site design. I also wanted to ensure that our site was responsive, so I developed our project using Bootstrap. This allowed us to create robust, responsive designs with relative ease.

Milestone One

SIDE BY SIDE OF WIREFRAME OF FORM VS ACTUAL FORM BY THE END OF THE MILESTONE

In the first milestone, our priorities were on creating a functional registration form and user dashboard. In order to get a baseline understanding of what data we needed to collect, we examined the registration form that was currently being used. We then drew out a wireframe for the form, structured out our databases, and connected them with the developed webform. After running a few tests and patching a few bugs, our form was about as done as we could get it.

We then shifted our focus to account authentication and population of the dashboard. While Matt and Carter worked on the authentication, I teamed up with Zachary to develop the dashboard. We created a multi-sectioned page with alloted containers for features we planned to execute later in the semester. Some of those features were never touched on, though, due to time constraints. However, we did ensure that the core features — viewing/editing team information, viewing/editing personal information, account management, etc. — were fleshed out to the fullest extent we could get them.
